# Kimikea Connect Order セットアップ手順

作成日: 2026-07-08

## 方針

Googleフォームで複雑な分岐を作るのではなく、Apps ScriptのWebアプリでカート形式の注文画面を作る。

最初はgrow専用で作る。
メーカー切り替え機能は作らない。

## 作成するスプレッドシート

スプレッドシート名:

`Kimikea Connect Order 管理表`

## 作成するシート

| シート名 | 役割 |
| --- | --- |
| 商品マスタ | grow商品のカテゴリー、カラー、仕入価格、販売価格、在庫、表示管理 |
| 加盟店マスタ | 加盟店ID、加盟店名、担当者、メールアドレス、表示管理 |
| 注文一覧 | 注文単位の合計情報、入金状況、発送状況を管理 |
| 注文詳細 | 注文内の商品明細、数量、単価、小計、利益を管理 |
| 設定 | 注文通知メール、振込先口座、支払期限を管理 |

## 商品マスタ

列:

- 商品カテゴリー
- カラー
- 仕入価格
- 販売価格
- 在庫
- 表示

初期データとしてgrow仕様のカラー一覧を登録する。

Webアプリに表示する商品は `表示` を TRUE にする。

## 加盟店マスタ

列:

- 加盟店ID
- 加盟店名
- 担当者名
- メールアドレス
- 電話番号
- 住所
- 表示

使い方:

- 注文画面には `表示` が TRUE の加盟店だけ表示する
- 加盟店を選ぶと、担当者名とメールアドレスが自動で入る
- 加盟店控えメールは、ここに登録したメールアドレスへ送信する

## 注文一覧

列:

- 注文番号
- 注文日時
- 加盟店ID
- 加盟店名
- 担当者名
- メールアドレス
- 合計袋数
- 送料
- 商品合計
- 請求合計
- 入金状況
- 発送状況
- 備考

## 注文詳細

列:

- 注文番号
- 商品カテゴリー
- カラー
- 数量
- 単価
- 小計
- 仕入単価
- 利益

## 送料ルール

- 注文単位は袋
- 1袋＝40本
- 1回の注文合計が5袋以上なら送料無料
- 1回の注文合計が4袋以下なら送料1,000円
- 4袋以下の場合、Webアプリ画面に `あと○袋で送料無料です` と表示する

## Apps Script設定手順

1. Googleスプレッドシートを新規作成する
2. スプレッドシート名を `Kimikea Connect Order 管理表` にする
3. 上部メニューの「拡張機能」を押す
4. 「Apps Script」を押す
5. `コード.gs` の中身を削除する
6. `Code.gs` の内容をすべて貼り付ける
7. 左側の「＋」からHTMLファイルを追加する
8. HTMLファイル名を `Index` にする
9. `Index.html` の内容をすべて貼り付ける
10. 保存する
11. 関数選択で `setupKimikeaConnectOrder` を選ぶ
12. 「実行」を押す
13. 権限確認が出たら許可する
14. スプレッドシートに戻る

完了すると、以下の5シートが作成される。

- 商品マスタ
- 加盟店マスタ
- 注文一覧
- 注文詳細
- 設定

## Webアプリ公開手順

1. Apps Script画面右上の「デプロイ」を押す
2. 「新しいデプロイ」を押す
3. 種類で「ウェブアプリ」を選ぶ
4. 説明に `Kimikea Connect Order MVP` と入力する
5. 実行ユーザーは「自分」
6. アクセスできるユーザーは、まずテスト用にBOSSが使える範囲で設定する
7. デプロイする
8. 表示されたWebアプリURLを開く

## このコードが何をしているのか

`Code.gs` は、スプレッドシートに管理シートを作り、Webアプリから届いた注文を保存するコード。

主な処理:

- `setupKimikeaConnectOrder`
  - スプレッドシート名を設定する
  - 商品マスタ、加盟店マスタ、注文一覧、注文詳細、設定を作成する
  - 入金状況、発送状況のプルダウンを作る

- `doGet`
  - Webアプリ画面を表示する

- `getVisibleProducts`
  - 商品マスタで `表示` がTRUEになっている商品だけをWebアプリへ渡す

- `getVisibleFranchises`
  - 加盟店マスタで `表示` がTRUEになっている加盟店だけをWebアプリへ渡す

- `submitCartOrder`
  - カート注文を受け取る
  - 加盟店マスタから加盟店情報を取得する
  - 注文番号を自動発行する
  - 送料を判定する
  - 注文一覧へ合計情報を保存する
  - 注文詳細へ商品ごとの明細を保存する
  - 利益を自動計算する

## 実行すると何が起きるのか

`setupKimikeaConnectOrder` を実行すると、スプレッドシートに注文管理の土台が作られる。

Webアプリから注文すると、以下のように保存される。

- 注文一覧に1注文1行で保存
- 注文詳細に商品ごとの明細を保存
- 5袋以上なら送料0円
- 4袋以下なら送料1,000円
- 商品ごとの利益を計算
- 設定シートと加盟店マスタの内容に応じて注文通知メールを送信
- 注文完了時に請求書PDFを作成し、加盟店メールへ添付
- 支払期限は注文日から設定シートの指定日数後として自動計算
- 在庫の自動減算・在庫不足判定は行わない

## 請求書PDFの設定

`setupKimikeaConnectOrder` を再実行すると、既存データを消さずに設定シートへ次の項目を追加する。

- 振込先銀行名
- 振込先支店名
- 口座種別
- 口座番号
- 口座名義
- 支払期限日数
- 送料
- 送料無料条件
- 送信元名
- 請求書ロゴファイルID

注文テストの前に、振込先の5項目をすべて入力する。

初期値として次の振込先が設定される。

- 静岡銀行
- 富士駅南支店
- 普通 0636210
- トクヒ）シールエクステキョウカイキミケア

KimikeaロゴをGoogleドライブへアップロードし、共有URLまたはファイルIDを `請求書ロゴファイルID` に入力する。ロゴ未設定時は、請求書左上に文字ロゴを表示する。PNGまたはJPGを推奨する。

請求書PDFの作成ではGoogleドキュメントとGoogleドライブを使用するため、初回実行時に追加のアクセス許可が表示される。

注文テストの前にApps Scriptの関数一覧から `authorizeDocumentApp` を選び、手動実行する。GoogleドキュメントとGoogleドライブの権限を承認すると、認証テスト用ドキュメントが作成され、すぐにゴミ箱へ移動する。

## 加盟店ログインの設定

`setupKimikeaConnectOrder` を再実行すると、加盟店マスタの `表示` 列の前へ `パスワード` 列を追加する。既存の加盟店データは削除しない。

各加盟店へ次を設定する。

- 加盟店ID
- 加盟店名
- 担当者名（`担当者` でも可）
- メールアドレス
- 電話番号（`電話` でも可）
- 住所
- パスワード
- 表示

`表示` がTRUEで、加盟店IDとパスワードが一致する加盟店だけログインできる。パスワードは加盟店ごとに異なる推測されにくい文字列を設定する。

ログイン状態は最長6時間で、操作するたびに有効期限を更新する。

## ホーム画面への追加

設定シートの `請求書ロゴファイルID` に登録したロゴを、起動画面とホーム画面用アイコンにも利用する。未設定の場合は黒・ゴールドの `K` アイコンを自動生成する。

- iPhone: Safariの共有ボタンから「ホーム画面に追加」
- Android: Chromeのメニューから「ホーム画面に追加」または「アプリをインストール」

Apps ScriptのHTML Serviceはiframe内で動作し、Content Serviceも一時URLへリダイレクトされるため、サービスワーカーによる完全なオフラインPWAは使用しない。現段階ではホーム画面追加・起動画面・ログイン保持・アプリ風3タブUIを提供する。

`setupKimikeaConnectOrder` は発送通知用の編集トリガーも自動作成する。注文一覧の発送状況を `発送済` に変更すると加盟店へ発送メールを送り、`発送通知日時` を記録する。同じ注文には重複送信しない。

送料と送料無料条件を設定シートで変更すると、Web注文画面・注文保存・請求書PDFへ共通して反映される。

## 注意点

- Googleフォームは一旦停止する
- メーカー選択機能は作らない
- 商品カテゴリーを選ぶと、そのカテゴリーのカラーだけ表示する
- 加盟店マスタの `表示` がFALSEの加盟店はWebアプリに表示しない
- 販売価格が未入力の商品はWebアプリに表示しない
- `表示` がFALSEの商品はWebアプリに表示しない
- growの正式販売価格、送料条件、在庫運用が決まったら商品マスタを更新する
