# Kimikea Connect Order スプレッドシート設計

作成日: 2026-07-08

## 基本方針

Kimikea Connect Orderは、grow専用のカート形式Webアプリとして作る。

スプレッドシートはデータベースとして使う。
Googleフォームは一旦停止し、Apps Script Webアプリから注文データを登録する。

## スプレッドシート名

`Kimikea Connect Order 管理表`

## シート構成

### 1. 商品マスタ

| 列名 | 内容 |
| --- | --- |
| 商品カテゴリー | ダークカラー、ライトカラー、原色 |
| カラー | 商品カラー |
| 仕入価格 | 協会の仕入単価 |
| 販売価格 | 加盟店への販売単価 |
| 在庫 | 在庫数または管理メモ |
| 表示 | Webアプリに表示する場合はTRUE |

### 2. 加盟店マスタ

| 列名 | 内容 |
| --- | --- |
| 加盟店ID | 加盟店を識別するID |
| 加盟店名 | 注文画面に表示する加盟店名 |
| 担当者名 | 加盟店を選択した時に自動反映する担当者 |
| メールアドレス | 加盟店控えメールの送信先 |
| 電話番号 | 連絡先 |
| 住所 | 発送先・管理メモ |
| 表示 | Webアプリに表示する場合はTRUE |

### 3. 注文一覧

| 列名 | 内容 |
| --- | --- |
| 注文番号 | 自動発行する注文ID |
| 注文日時 | Webアプリから注文された日時 |
| 加盟店ID | 加盟店マスタとひも付けるID |
| 加盟店名 | 注文した加盟店名 |
| 担当者名 | 注文担当者 |
| メールアドレス | 加盟店控えメールの送信先 |
| 合計袋数 | 注文合計袋数 |
| 送料 | 5袋以上0円、4袋以下1,000円 |
| 商品合計 | 商品代金の合計 |
| 請求合計 | 商品合計 + 送料 |
| 入金状況 | 未入金、入金済 |
| 発送状況 | 未発送、準備中、発送済、完了、キャンセル |
| 備考 | 注文メモ |

### 4. 注文詳細

| 列名 | 内容 |
| --- | --- |
| 注文番号 | 注文一覧とひも付ける注文ID |
| 商品カテゴリー | 注文商品のカテゴリー |
| カラー | 注文商品のカラー |
| 数量 | 注文袋数 |
| 単価 | 加盟店販売単価 |
| 小計 | 単価 × 数量 |
| 仕入単価 | 協会仕入単価 |
| 利益 | 小計 - 仕入単価 × 数量 |

## 保存の考え方

- 注文一覧は「1注文につき1行」
- 注文詳細は「1商品明細につき1行」
- 加盟店情報は加盟店マスタを正とし、注文時点の加盟店ID、加盟店名、担当者名、メールアドレスを注文一覧へ保存する
- 同じ注文番号で注文一覧と注文詳細をつなぐ
- 月別集計、加盟店別集計、商品別集計は注文一覧と注文詳細から後で作る

## MVPで優先すること

- grow商品の注文
- 商品カテゴリー管理
- 商品カテゴリー別カラー表示
- 商品一覧・数量入力
- 送料判定
- 利益計算
- 注文履歴
- 発送状況管理
- 入金状況管理
- 注文番号の自動発行
- 加盟店マスタからの加盟店選択と加盟店控えメール送信
- Webアプリ上での注文受付
