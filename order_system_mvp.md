# Kimikea Connect Order MVP

作成日: 2026-07-03
更新日: 2026-07-09

## 目的

Kimikea Connect Orderの最初の開発範囲は、加盟店・コーディネーターからのgrowシールエクステ注文を管理するカート形式Webアプリ。

Googleフォームは一旦停止し、Googleスプレッドシートをデータベース、Google Apps ScriptをWebアプリとして使う。

## 設計方針

MVPはgrow専用の注文システムとして完成度を高める。

- 仕入先はgrow固定
- メーカー切り替え機能は作らない
- Googleフォームで複雑な分岐は作らない
- Apps ScriptのWebアプリでカート形式の注文画面を作る
- 商品カテゴリー、カラー、数量を選んで注文できるようにする
- スプレッドシートは `商品マスタ`、`加盟店マスタ`、`注文一覧`、`注文詳細`、`設定` の5シートを土台にする
- 将来、特別な理由があれば他メーカー対応できるよう、商品マスタ構造には拡張余地を残す

## スプレッドシート名

`Kimikea Connect Order 管理表`

## 作成するシート

| シート | 役割 |
| --- | --- |
| 商品マスタ | grow商品のカテゴリー、カラー、仕入価格、販売価格、在庫、表示を管理 |
| 加盟店マスタ | 加盟店ID、加盟店名、担当者、メールアドレス、表示を管理 |
| 注文一覧 | 1注文ごとの合計、入金状況、発送状況を管理 |
| 注文詳細 | 注文内の商品明細、数量、単価、利益を管理 |
| 設定 | 注文通知メールの送信先、送信ON/OFFを管理 |

- 商品カテゴリー管理
- 商品カテゴリー選択後、そのカテゴリー内のカラーだけを表示
- 商品一覧・数量入力
- カート形式の注文
- 送料判定
- 利益計算
- 注文履歴
- 発送状況管理
- 入金状況管理
- 注文番号の自動発行

- 加盟店マスタから表示中の加盟店だけを注文画面に表示
- 加盟店選択時の担当者名・メールアドレス自動反映
- 注文通知メールと加盟店控えメールの送信

## 計算ルール

```text
注文単位 = 袋
1袋 = 40本
合計袋数 = 注文詳細の数量合計
商品合計 = 注文詳細の小計合計
送料 = 1回の注文合計が5袋以上なら0円、4袋以下なら1,000円
請求合計 = 商品合計 + 送料
注文詳細の利益 = (単価 - 仕入単価) × 数量
```

送料は加盟店から預かって発送費用に充てる実費扱いのため、利益には含めない。

## 入金状況

- 未入金
- 入金済

## 発送状況

- 未発送
- 準備中
- 発送済
- 完了
- キャンセル

## 運用フロー

1. 協会側が商品マスタにgrow商品を登録する
2. Webアプリで表示したい商品の `表示` をTRUEにする
3. 協会側が加盟店マスタに加盟店情報を登録する
4. Webアプリで表示したい加盟店の `表示` をTRUEにする
5. 加盟店がWebアプリで加盟店名を選択する
6. 担当者名とメールアドレスが自動反映される
7. 加盟店が商品をカートに入れて注文する
8. Apps Scriptが注文番号を自動発行する
9. 注文一覧へ合計情報を保存する
10. 注文詳細へ商品明細を保存する
11. 設定シートと加盟店マスタに応じて注文通知メールを送信する
12. 協会側が入金状況と発送状況を更新する

- Webアプリで加盟店を選ぶと担当者名とメールアドレスが自動反映される
- カートに商品を追加できる
- 注文番号が自動発行される
- 注文一覧に合計情報が保存される
- 注文詳細に商品明細が保存される
- 5袋以上は送料無料、4袋以下は送料1,000円で計算される
- 入金状況と発送状況を管理できる
- 注文通知メールと加盟店控えメールを送信できる
