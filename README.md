# Kimikea Connect

NPO法人シールエクステ協会きみけあの会員向け仕事アプリ。

Kimikea Connect は、注文サイトではない。

加盟店・コーディネーター・本部が毎日開く「美容師専用OS」として設計する。

注文、スタイル共有、教育、情報共有、加盟店検索、売れる提案を一つにまとめる。

## 正式公開URL

加盟店・コーディネーターへ案内するKimikea Connectの入口URLは、今後ずっと以下に統一する。

[https://boss-team1129.github.io/Kimikea-Connect/](https://boss-team1129.github.io/Kimikea-Connect/)

- 本番公開は `boss-team1129/Kimikea-Connect` の `main` ブランチからGitHub Pagesで行う
- 新しいリポジトリや別フォルダで公開URLを増やさない
- Apps ScriptのWebアプリURLは裏側のAPIとして使用し、加盟店には案内しない
- テスト中のURLや確認用URLは加盟店へ案内しない
- キャッシュ対策は公開URLを変えず、ファイルのバージョン更新やキャッシュ更新で対応する
- 今後の更新でも、既存のログイン情報、投稿データ、注文履歴、保存データを削除しない

## 目的

- 加盟店・コーディネーターが毎日の仕事で使える
- 発注を簡単にする

## 第1段階

Googleスプレッドシート、Google Apps Script、GitHub Pagesを使って無料または低コストで運用する。

Googleフォームは一旦停止し、アプリ内の発注センターとしてWebアプリのカート形式注文画面へ移行する。

## MVPの設計方針

Kimikea ConnectのMVPは、会員向け仕事アプリとして設計する。

- トップページは協会のホーム画面として作る
- Style Bookは美容師専用レシピ図鑑として作る
- Orderは発注センターとして作る
- Apps ScriptのWebアプリでカート形式の発注画面を作る
- Googleフォームで複雑な分岐は作らない
- 注文画面でメーカー選択は行わない
- 商品マスタはgrow商品を中心に管理する
- 仕入先は内部的にgrow固定で扱う
- 将来、特別な理由があれば他メーカー対応も可能な余白は残す
- 現時点ではメーカー切り替え機能、メーカー選択機能は実装対象外

## 注意

- Googleフォームは一旦停止し、Webアプリ注文へ移行する
- Apps Scriptは商品マスタ、加盟店マスタ、注文一覧、注文詳細、設定の管理表を作成する
- 商品マスタの金額は、新しくWebアプリから受け付ける注文に反映される
- 加盟店控えメールは加盟店マスタのメールアドレスへ送信する
- 注文完了時に請求書PDFを自動作成し、加盟店控えメールへ添付する
- 振込先口座と支払期限日数は設定シートで変更できる
- 請求書はA4で、Kimikeaロゴ・法人情報・明細・振込先を表示する
- ロゴは設定シートのGoogleドライブファイルIDから読み込む
- growへ都度注文するため、在庫の自動減算・不足通知は行わない
- 入金状況は `未入金`・`入金済`、発送状況は `発送準備`・`発送済` のプルダウンで管理する
- 発送状況を `発送済` に変更すると、加盟店へ発送通知メールを一度だけ送信する
- 送料・送料無料条件・送信元名は設定シートから変更できる
- PCでは従来の横並び・表形式を維持し、スマホでは入力欄を縦並び、カートをカード形式で表示する
- スマホではボタン、金額、送料無料までの残り袋数を大きく表示する
- 加盟店IDとパスワードでログインし、表示TRUEの加盟店だけ利用できる
- ログイン後は加盟店名・担当者・メール・電話・住所を本人分だけ表示する
- 注文履歴はログイン中の加盟店IDに一致する注文だけ表示する
- 注文番号から商品明細を展開し、請求書PDFを再表示できる
- 発送状況は `発送準備`・`発送済`・`キャンセル` の3種類で管理する
- 入金管理は使用せず、growへの発注は注文受付後すぐ行う運用とする
- ホーム画面追加用メタ情報、動的アプリアイコン、起動画面を表示する
- ログイン用セッショントークンを端末へ保持し、サーバー側の有効期限内は再ログインを省略する
- `新規注文`・`注文履歴`・`お知らせ` の3タブで構成する
- Apps Scriptのiframe・配信URL制約により、オフライン用サービスワーカーは実装対象外とする
- growのダークカラー、ライトカラー、原色はBOSS指定の仕入価格・販売価格で管理する
- BRIDGEは過去・参考データとして扱い、MVPの実装対象から外す
- Kimikea Connect関連作業では、`development_progress.md` に従って進捗をパーセンテージで表示する
- 今後追加するページは独立サイトではなく、Kimikea Connectアプリ内機能として設計する
